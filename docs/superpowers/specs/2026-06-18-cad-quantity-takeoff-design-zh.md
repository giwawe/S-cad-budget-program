# CAD 装修算量设计文档

日期：2026-06-18

## 目标

建设第一版基于 CAD 的装修算量工具。

输入是 DWG 平面方案图。输出是一个可修改的算量表，准确度要足够作为后续装修报价的依据。第一版聚焦算量，不直接生成最终装修报价。

准确性优先于完全自动化。为了提高系统识别准确率，可以要求设计师对 CAD 图纸做少量规范化处理。

## 产品方向

采用“轻量 CAD 制图规范 + 自动识别”的方式。

系统应自动读取规范图层，生成初始算量表，只在异常或不确定数据处要求用户处理。预期流程如下：

1. 设计师按要求准备带有 `QUOTE_*` 图层的 DWG 或 DXF。
2. 用户上传 DWG 或 DXF。
3. DXF 直接解析；DWG 先自动转换为 DXF，转换失败时提示设计师另存 DXF 后重新上传。
4. 系统解析空间边界、空间名称、墙体、窗、开放边界、门洞、楼层和层高。
5. 系统生成可修改算量表。
6. 用户只复核异常项或默认推断值。
7. 已确认的算量数据导出，后续作为报价输入。

## CAD Adapter 输入方案

第一版采用 `DXF 直读 + DWG 自动转换 + 转换失败兜底提示另存 DXF`。

系统边界应保持清晰：

```text
DWG / DXF 文件 -> CAD Adapter -> ProjectInput JSON -> 算量引擎 -> 可修改 Excel / JSON 表格
```

CAD Adapter 只负责把 CAD 文件翻译为标准 `ProjectInput` 数据，不直接承载报价公式和最终报价逻辑。这样可以让 CAD 解析、算量公式和后续报价模块分别演进。

文件入口规则：

```text
DXF
- 直接使用 DXF 解析器读取。
- 解析结果生成 ProjectInput JSON。

DWG
- 后台调用转换工具转换为 DXF。
- 转换后的 DXF 复用同一套解析逻辑。
- 转换失败时阻断本次解析，并提示设计师在 CAD 中另存为 DXF 后重新上传。
```

系统应保留以下追溯材料：

```text
原始上传文件
转换后的 DXF 文件
CAD Adapter 解析日志
生成的 ProjectInput JSON
异常清单
```

CAD 单位处理采用“双校验”：

1. 系统读取 DXF/DWG 文件单位。
2. 上传时要求用户确认项目单位。
3. 如果文件单位和用户确认单位冲突，阻断或标记为严重异常，避免整体面积比例错误。

## CAD 图层规范

### 必需核心图层

`QUOTE_ROOM`

空间闭合边界。每个房间或空间应有一条闭合多段线。它是地面面积、空间归属和地面周长的主要依据。

`QUOTE_WINDOW`

窗户标记。系统优先从块属性中读取窗宽和窗高。如果缺少窗高，系统使用默认窗高，并将该值标记为默认推断。

第一版同时支持现有窗户画法：`QUOTE_WINDOW` 图层上的封闭窗洞轮廓，轮廓可以是矩形、多边形或带弧线的闭合图形，内部可带线段作为窗符号特征。系统应识别封闭轮廓，优先从块属性或标注读取窗宽；没有属性时，从封闭轮廓的主方向投影长度或边界展开长度推断窗宽，再结合窗高计算窗面积。

### 可选核心图层

`QUOTE_TEXT`

空间名称文字。如果存在该图层，系统优先使用它作为空间名称来源。如果不存在，系统应尝试读取 `QUOTE_ROOM` 边界内已有的普通 CAD 文字。

空间名称优先级：

1. 块属性或规范标注。
2. `QUOTE_TEXT`。
3. `QUOTE_ROOM` 边界内普通 CAD 文字。
4. 表格中人工补录。

`QUOTE_DOOR`

门洞标记。第一版读取并保存门洞数据，但默认不从墙面面积中扣除门洞面积。门可以使用现有模型或块；系统优先读取块名、插入点、旋转、缩放和属性，缺少属性时从几何关系推断门洞宽度。

### 推荐准确性图层

`QUOTE_WALL`

实际墙体或可施工墙面几何。用于计算墙面计量周长，并避免把开放空间边界错误计入墙面面积。

`QUOTE_OPENING`

开放边界或非墙体边界。用于客餐厅等开放空间：`QUOTE_ROOM` 为了计算地面面积必须闭合，但其中一部分边界并不是真实墙面，不应计入乳胶漆等墙面施工面积。

### 多楼层图层

`QUOTE_FLOOR`

楼层标记，用于别墅和多层项目。

`QUOTE_HEIGHT`

可选层高标记。如果存在，它优先于楼层默认层高和项目默认层高。

## 层高规则

层高优先级：

1. 人工修改后的空间层高。
2. `QUOTE_HEIGHT` 标记。
3. 楼层默认层高。
4. 项目默认层高。

商品房通常可以使用一个项目默认层高。别墅项目应支持每层设置默认层高，并允许对特殊空间单独覆盖。

## 开放空间墙面算量

地面边界和墙面计量边界是两个不同概念。

`QUOTE_ROOM` 始终用于地面面积和地面周长，不能直接把它当作墙面计量周长。

对于客厅、餐厅等开放空间：

```text
地面面积 = QUOTE_ROOM 闭合边界面积
地面周长 = QUOTE_ROOM 闭合边界周长
墙面计量周长 = 空间边界中实际有墙或可施工墙面的部分
```

墙面面积公式：

```text
墙面毛面积 = 墙面计量周长 * 层高
墙面净面积 = 墙面毛面积 - 窗面积
```

第一版记录门洞面积，但默认不扣除门洞面积。

墙面计量周长识别优先级：

1. 如果边界段被 `QUOTE_OPENING` 标记，则不计入墙面。
2. 如果 `QUOTE_ROOM` 的边界段靠近或重合 `QUOTE_WALL`，则计入墙面。
3. 如果既没有匹配到墙体，也没有开放边界标记，则该边界段标记为不确定。
4. 用户可以在表格中修改墙面计量周长，后续也可在图纸复核界面中修改。

算量表必须同时展示地面周长和墙面计量周长，避免混淆。

## 特殊空间规则

第一版应通过轻量 CAD 规范和明确的空间类型来支持特殊空间，不尝试从任意复杂几何中自动推断建筑语义。

推荐空间类型：

```text
normal
void
void_opening
stair
stair_hall
balcony
terrace
elevator_shaft
```

空间类型可以来自 CAD 对象属性、空间名称约定、表格修改，或必要时来自专用标记图层。

### 挑空 / 跃层空间

别墅项目常见挑空客厅或其他跨层空间。

推荐 CAD 支持：

```text
QUOTE_VOID
```

`QUOTE_VOID` 用于标记挑空区域或楼板洞口，可与 `QUOTE_ROOM` 边界关联。

计算规则：

```text
地面面积 = 一层 QUOTE_ROOM 边界面积
地面周长 = 一层 QUOTE_ROOM 边界周长
墙面计量周长 = 基于 QUOTE_WALL / QUOTE_OPENING 判断的墙面周长
墙面毛面积 = 墙面计量周长 * 挑空实际高度
墙面净面积 = 墙面毛面积 - 窗面积
```

挑空高度优先级：

1. 人工修改后的空间高度。
2. `QUOTE_HEIGHT` 标记。
3. `QUOTE_VOID` 高度属性。
4. 关联楼层默认层高之和。
5. 项目默认层高。

如果上层图纸中同一挑空区域表现为楼板洞口，则该上层洞口不能当作普通房间，也不能生成地面面积。它可以作为辅助的 `void_opening` 行显示，用于复核。

### 楼梯空间和楼梯过道

楼梯相关空间应明确标记：

```text
space_type = stair
space_type = stair_hall
```

规则：

```text
楼梯空间
- 记录地面面积，但标记为特殊面积。
- 使用 QUOTE_WALL / QUOTE_OPENING 和对应高度计算墙面面积。
- 第一版不自动计算楼梯踏步、踢面、斜板、扶手或楼梯饰面展开工程量。
- 后续为楼梯专项工程量预留手工录入字段。

楼梯过道
- 如果是普通平层过道，按普通空间计算。
- 如果连接挑空或楼梯井，允许开放边界和不确定墙段。
```

### 阳台和露台空间

阳台和露台应明确标记：

```text
space_type = balcony
space_type = terrace
```

规则：

```text
阳台
- 计算地面面积。
- 只计算实际墙体段的墙面面积。
- 栏杆或开放边不计入室内墙面面积。
- 如果封窗，使用 QUOTE_WINDOW 并扣除窗面积。

露台
- 计算地面面积。
- 通常只按需要计算相邻建筑外墙或女儿墙。
- 开放边不计入室内墙面乳胶漆面积。
- 通过表格字段控制是否计入室内装修算量。
```

表格应提供：

```text
是否室外空间
是否计入室内地面算量
是否计入室内墙面乳胶漆算量
```

### 电梯井

电梯井应明确标记：

```text
space_type = elevator_shaft
```

规则：

```text
- 默认排除在装修算量之外。
- 不计地面面积。
- 不计墙面面积。
- 保留为辅助行或校验对象，避免被系统误识别为普通房间。
```

### 外墙面积

室内墙面算量和外立面/外墙算量应分开。

第一版可通过轻量专用图层支持外墙面积：

```text
QUOTE_EXT_WALL
QUOTE_EXT_OPENING
```

规则：

```text
外墙毛面积 = 外墙计量长度 * 外墙高度
外墙净面积 = 外墙毛面积 - 外墙洞口面积
```

外墙工程量应显示在单独的外墙/外立面表中，不应混入房间空间行。第一版不尝试自动处理复杂外立面建模、保温系统、真石漆/涂料构造或装饰造型。

当用户修改可编辑输入时，相关计算字段必须自动重新计算。

## 数据状态

每个关键数据都应有来源或状态，方便审计和复核。

状态包括：

`confirmed`

来自规范 CAD 图层且数据完整，或已经人工确认。

`default_inferred`

系统使用了默认值，例如默认窗高。

`needs_review`

系统检测到数据缺失、冲突或不确定。

`manually_edited`

用户修改过系统生成值。

后续生成正式报价时，应只使用已确认或已人工接受的数据。

## 异常分级

解析结果采用分级处理，避免系统悄悄算错。

严重问题会阻断生成正式算量表：

```text
DWG 转 DXF 失败
CAD 单位无法确认或与用户确认单位冲突
关键图层缺失
空间边界未闭合
空间面积为 0 或明显异常
多楼层项目缺少必要楼层信息
```

轻微问题继续生成表格，但必须在对应行和异常汇总中标记为需确认：

```text
窗高缺失，使用默认窗高
空间名称来自普通 CAD 文字且置信度较低
门模型没有属性，使用几何推断宽度
窗户封闭轮廓匹配墙体存在轻微偏差
墙面计量边界存在不确定段
特殊空间类型来自名称推断而非明确属性
```

算量表应包含数据来源、置信度、是否需确认和问题说明，后续报价默认只使用 `confirmed`、`manually_edited` 或用户已接受的数据。

## 识别和计算规则

空间识别：

1. 从 `QUOTE_ROOM` 读取闭合多段线。
2. 校验每个空间边界是否闭合。
3. 根据闭合边界计算地面面积和地面周长。
4. 优先从 `QUOTE_TEXT` 匹配空间名称，没有则读取边界内普通 CAD 文字。
5. 检测空间无名称、名称重复、空间重叠和无效边界。

窗户识别：

1. 读取 `QUOTE_WINDOW`。
2. 优先使用块属性中的宽度和高度。
3. 如果不是块属性窗，则识别 `QUOTE_WINDOW` 上的封闭窗洞轮廓；轮廓可以是矩形、多边形或带弧线的闭合图形，内部线段可作为窗符号辅助特征。
4. 对普通矩形窗，使用长边推断窗宽；对多边形或带弧线窗，使用主方向投影长度或与墙体重合/贴近的边界展开长度推断窗宽。
5. 如果缺少高度，使用默认窗高，并标记为 `default_inferred`。
6. 将窗户匹配到空间边界或墙段。
7. 计算窗面积，并从墙面面积中扣除。

门洞识别：

1. 读取 `QUOTE_DOOR`。
2. 优先读取现有门模型或块的块名、插入点、旋转、缩放和属性。
3. 如果缺少属性，则从门洞两端距离或门模型几何推断门宽。
4. 如果有门洞数量和门洞面积，则保存。
5. 第一版默认不从墙面面积中扣除门洞面积。

墙面识别：

1. 读取 `QUOTE_WALL`。
2. 对比 `QUOTE_ROOM` 边界段与附近 `QUOTE_WALL` 的关系。
3. 排除 `QUOTE_OPENING` 标记的开放边界。
4. 生成墙面计量周长。
5. 将不确定墙面边界标记为异常。

## 异常清单

系统应报告以下异常：

```text
空间边界未闭合
空间边界与其他空间重叠
空间没有名称
一个空间内存在多个名称
窗户无法归属到空间或墙体
窗户缺少高度，已使用默认高度
门洞无法归属到空间或墙体
墙面计量边界不确定
QUOTE_OPENING 与 QUOTE_WALL 冲突
挑空高度缺失或不确定
上层挑空洞口被误识别为普通房间
楼梯空间需要手工录入楼梯专项工程量
阳台或露台开放边界不确定
电梯井未标记为排除
外墙高度或洞口数据缺失
多层项目缺少楼层信息
层高缺失，已使用默认层高
CAD 单位或比例不明确
```

异常应同时显示在对应表格行内，以及单独的异常汇总中。

## 系统模块

### DWG 导入模块

上传 DWG 文件，并将其转换成系统可解析的中间格式，可能是 DXF 或 JSON。DWG 是封闭格式，实现时应避免手写 DWG 解析器。

### CAD 规范识别模块

读取 `QUOTE_*` 图层，并将 CAD 对象转换为结构化实体：

```text
RoomBoundary
RoomText
WindowMarker
DoorMarker
WallGeometry
OpeningGeometry
FloorMarker
HeightMarker
VoidMarker
ExteriorWallGeometry
ExteriorOpeningGeometry
```

### 几何匹配模块

将文字、窗、门洞、墙体、开放边界、楼层和层高匹配到空间。

该模块也负责几何校验并生成异常。

### 算量引擎

计算地面面积、地面周长、墙面计量周长、墙面毛面积、窗面积、门洞面积和墙面净面积。

算量引擎应以公式驱动，并与 CAD 解析逻辑分离。

### 可修改表格模块

展示生成的算量表、数据来源、状态和异常。

允许用户修改数据，并自动重新计算相关字段。

### 导出模块

将已确认的算量表导出为 Excel，作为后续报价输入。

## 第一版范围

包含：

```text
DWG 导入和转换
QUOTE_ROOM 解析
QUOTE_TEXT 和普通 CAD 文字兜底解析
QUOTE_WINDOW 解析
QUOTE_DOOR 解析，保存但默认不扣除
QUOTE_WALL 解析
QUOTE_OPENING 解析
QUOTE_VOID 解析，用于挑空空间和楼板洞口
楼梯、楼梯过道、阳台、露台、电梯井等特殊空间类型支持
可选 QUOTE_EXT_WALL 和 QUOTE_EXT_OPENING 解析，用于外墙工程量
项目默认层高
楼层默认层高
空间层高覆盖
可修改算量表
异常汇总
Excel 导出
```

不包含：

```text
最终装修报价生成
对任意非规范 DWG 的机器学习识别
没有 QUOTE_ROOM 时自动推断空间边界
默认扣除门洞面积
自动计算楼梯踏步、踢面、斜板等楼梯专项工程量
自动处理复杂外立面建模或外墙饰面系统
完整兼容所有 CAD 制图风格
```

## 成功标准

第一版满足以下条件即视为成功：

1. 符合 `QUOTE_*` 规范的 DWG 可以生成每个空间一行的可修改表格。
2. 地面面积根据 `QUOTE_ROOM` 计算。
3. 开放空间的墙面面积不会错误计入开放边界。
4. 窗面积会被扣除；有块属性时读取窗高，没有时使用默认窗高。
5. 门洞数据会被保留，但默认不影响墙面面积。
6. 挑空空间可以使用明确高度，并且不会重复生成上层普通房间面积。
7. 楼梯、阳台、露台、电梯井等特殊空间可以被标记，并按第一版规则处理。
8. 如启用外墙算量，外墙工程量与室内空间工程量分开显示。
9. 用户可以在导出前识别并处理不确定数据。
10. 导出的 Excel 表可以作为后续报价模块的输入。
