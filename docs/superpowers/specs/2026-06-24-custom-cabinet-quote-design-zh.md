# 全屋定制与橱柜 CAD 标识报价设计

## 背景

商品房整装报价已能从算量结果自动生成空间项目、湿区项目、瓷砖片数、窗帘、门类、淋浴隔断、瓷砖加工费等项目。真实样例中 `全屋定制` 和 `橱柜` 仍为模板默认，原因是当前 CAD/算量结果没有柜体标识来源。

本阶段目标是新增轻量 CAD 标识图层，让设计师可以在平面图中标出全屋定制和橱柜的长度与必要属性。系统只做稳定的总量自动化，不展开复杂定制报价明细。

## 范围

本阶段新增两类标识：

- `QUOTE_CUSTOM`：全屋定制，包括衣柜、餐边柜、鞋柜、玄关柜等。
- `QUOTE_CABINET`：橱柜，包括地柜、吊柜等。

本阶段不做：

- 不自动拆分 `主卧衣柜`、`餐边柜`、`鞋柜` 等模板报价行。
- 不计算门板、台面、玻璃门、拉手、抽屉、灯带、拉直器、功能五金等细项。
- 不把全屋定制或橱柜反向写回 CAD。
- 不做 `INSERT` 块导入，先只支持 `LINE` / `LWPOLYLINE`，降低出图规范复杂度。

## CAD 出图口径

### `QUOTE_CUSTOM`

设计师在柜体投影长度方向画 `LINE` 或 `LWPOLYLINE`：

- 线段长度表示柜体投影长度。
- 多段线长度表示连续柜体总投影长度。
- 闭合多段线允许导入，系统使用最小旋转矩形的最长边作为柜体投影长度，不使用闭合轮廓周长。

可选属性：

- `HEIGHT` / `height` / `高`：柜体投影高度，单位规则沿用门窗属性，大于 `20` 按毫米，小于等于 `20` 按米。
- `TYPE` / `type` / `类型`：柜体类型，例如 `衣柜`、`餐边柜`、`鞋柜`、`玄关柜`。
- `ROOM` / `room` / `空间`：人工指定空间名或空间 ID。缺失时系统按几何位置归属房间。

默认规则：

- 缺少高度时，报价高度默认 `2.6m`。
- 高度小于 `1.0m` 的全屋定制柜按长度报价，不按投影面积报价。
- 缺少类型不阻止报价；明细中保留空类型，复核备注提示类型待确认。

### `QUOTE_CABINET`

设计师在橱柜投影长度方向画 `LINE` 或 `LWPOLYLINE`：

- 线段或多段线长度表示橱柜长度。
- 地柜和吊柜在平面中可能重叠，不能仅靠几何判断类型。
- `TYPE` / `type` / `类型` 应填写 `地柜` 或 `吊柜`；缺失时仍导入，但报价复核备注提示地柜/吊柜需确认。

可选属性：

- `TYPE` / `type` / `类型`：`地柜`、`吊柜` 或其他设计师约定类型。
- `ROOM` / `room` / `空间`：人工指定空间名或空间 ID。缺失时系统按几何位置归属房间。

本阶段橱柜报价数量默认为所有 `QUOTE_CABINET` 标识长度汇总。后续如果模板拆分 `地柜` / `吊柜`，可按 `TYPE` 分别汇总。

## 数据模型

新增稳定模型：

- `FixtureMarker`
  - `id`
  - `layer`
  - `kind`: `custom` 或 `cabinet`
  - `points`
  - `length`
  - `height`
  - `height_defaulted`
  - `fixture_type`
  - `room_id`
  - `floor`
  - `attributes`

实现使用一个通用模型，减少重复几何解析逻辑；对外 JSON 字段分别暴露为 `custom_items` 和 `cabinet_items`，保持报价规则清晰。

`ProjectInput` 新增：

- `custom_items`
- `cabinet_items`

`QuantityResult` 新增：

- `custom_details`
- `cabinet_details`

明细字段需要保留 `room_id` / `room_name` / `length` / `height` / `height_defaulted` / `fixture_type`，供报价复核列和后续专项报价使用。

`custom_details` 还需要保留本条明细的计价口径：

- `pricing_mode="projected_area"`：高度大于等于 `1.0m`，按 `length * effective_height` 计入投影面积。
- `pricing_mode="length"`：高度小于 `1.0m`，按 `length` 计入长度。

## 算量与报价规则

### 全屋定制

模板项：`全屋定制`

数量分为两部分：

```text
projected_area = sum(length * effective_height for height >= 1.0m)
length_quantity = sum(length for height < 1.0m)
```

其中：

- `effective_height = HEIGHT`，如果标识提供高度。
- `effective_height = 2.6m`，如果缺少高度。
- 缺少高度时使用 `2.6m`，因此进入投影面积口径。

本阶段模板只有一个 `全屋定制` 行时，报价数量使用 `projected_area`，并在复核备注中写入高度小于 `1.0m` 的长度合计，提示预算员后续拆分或手工调整。后续如果模板新增低柜长度项目，可把 `length_quantity` 单独汇总到对应模板项。

复核状态：

- 全部有高度：`自动生成`
- 存在默认高度：`自动生成-默认推断`
- 备注：`全屋定制缺少高度时默认2.6m，需复核`
- 存在高度小于 `1.0m` 的低柜：追加 `高度小于1m的全屋定制按长度报价，需拆分确认`
- 存在缺失类型：追加 `全屋定制类型缺失，需确认柜体类型`

### 橱柜

模板项：`橱柜`

数量：

```text
sum(QUOTE_CABINET.length)
```

复核状态：

- 所有标识都有类型：`自动生成`
- 存在缺失类型：`自动生成-默认推断`
- 备注：`橱柜类型缺失，地柜/吊柜需确认`

## 房间归属

房间归属优先级：

1. 标识属性 `ROOM` 明确指定空间 ID 或空间名。
2. 标识线段中点落在某个房间内。
3. 标识线段中点靠近某个房间边界，使用与门窗类似的容差。
4. 无法归属时仍计入全屋汇总，但明细 `room_id=None`，复核备注提示空间归属待确认。

## 兼容性

- 旧 `ProjectInput` JSON 缺少新字段时仍可加载。
- 旧 `QuantityResult` JSON 缺少新明细时仍可报价；`全屋定制` 和 `橱柜` 保持模板默认。
- 规则 JSON 缺少新自动化列表时仍可加载，新规则字段默认空集合。
- CLI 命令保持兼容，不新增必填参数。

## 测试策略

需要覆盖：

- 模型默认字段向后兼容。
- DXF 导入 `QUOTE_CUSTOM` / `QUOTE_CABINET` 的 `LINE` 和 `LWPOLYLINE`。
- 属性解析：高度、类型、空间。
- 全屋定制缺高按 `2.6m` 默认推断。
- 全屋定制高度小于 `1.0m` 时进入长度计价口径，不计入投影面积。
- 橱柜地柜/吊柜重叠时按两条标识分别计长，不能因几何重叠去重。
- 报价中没有标识时保持模板默认。
- 报价中有标识时 `全屋定制` 按高度分流为投影面积和低柜长度口径，`橱柜` 按长度。
- 缺类型时复核备注明确提示。
- 更新 `docs/cad-lightweight-drawing-standard-zh.md`。

## 后续专项报价

本阶段明细中保留 `fixture_type` 和空间归属，是为了后续支持：

- 主卧衣柜、次卧衣柜、餐边柜、鞋柜、玄关柜等拆分报价。
- 橱柜地柜、吊柜分别报价。
- 台面、门板、玻璃门、五金、灯带等全屋定制专项清单。

这些不进入本阶段实现范围。
