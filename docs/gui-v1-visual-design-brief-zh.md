# GUI v1 视觉设计 Brief

## 设计目标

GUI v1 应该像一个安静、可靠的预算工作台，而不是营销页或炫技界面。用户打开后第一眼要知道：选文件、运行、看结果、打开输出。

关键词：

- 专业
- 清晰
- 稳定
- 密度适中
- 错误可理解

## 设计阶段如何使用 skill

开发阶段可以使用设计类 skill 辅助产出更好的界面方案，例如设计咨询、设计评审、界面 QA。它们适合在实现前和实现后使用：

- 实现前：产出布局、组件状态、视觉规范、QSS 方向。
- 实现后：检查间距、层级、文本溢出、状态表达和可读性。

交付后的 GUI 不应在运行时依赖 Codex skill、联网 AI 或当前开发环境。GUI 运行时只调用项目内 Python 服务函数。

## 推荐风格

- 桌面工具风格，不做 landing page。
- 主色使用低饱和蓝绿色或中性蓝，强调可靠和工程感。
- 背景使用浅灰或白色，避免大面积深色、渐变和装饰图形。
- 卡片只用于独立信息块，避免卡片套卡片。
- 表格和统计面板要适合反复查看，不要过度留白。
- 字体使用系统默认中文友好字体，Windows 优先 `Microsoft YaHei UI`。

## 页面布局建议

### 主窗口

建议尺寸：

- 默认：1200 x 760
- 最小：1000 x 640

布局：

- 左侧窄导航：运行、结果、设置。
- 右侧主区域：当前页面内容。
- 底部状态栏：当前状态、最近输出目录、耗时。

### 运行页

上半区是输入表单：

- DXF 文件
- 模板 Excel
- 单价表 Excel
- 输出目录

每行包含：

- 标签
- 路径输入框
- 文件选择按钮
- 清空或打开按钮

下半区是操作和日志：

- 主按钮：生成正式报价
- 次按钮：运行真实验收
- 次按钮：打开输出目录
- 日志输出框

按钮状态：

- 空闲：主按钮可用。
- 运行中：按钮禁用，显示进度或忙碌状态。
- 成功：状态栏显示绿色成功摘要。
- 失败：状态栏显示红色失败摘要，并在日志中显示失败阶段。

### 结果页

顶部统计卡片：

- 自动算量
- 自动汇总
- 模板默认
- 默认推断
- 异常提示
- 单价匹配行

中部复核行动表：

- 优先级
- 行动名称
- 影响行数
- 涉及项目
- 建议处理

底部输出文件：

- `quote-priced.xlsx`
- `quote-priced-review.md`
- `quote-priced-review-checklist.xlsx`
- `summary.json`

每个文件提供打开按钮。

## 组件规范

- 文件选择按钮使用文件夹图标。
- 运行按钮使用播放图标。
- 打开目录按钮使用外部打开或文件夹打开图标。
- 成功状态使用绿色小图标和简短文本。
- 失败状态使用红色小图标和清晰错误。
- high 行动使用红色标签，medium 使用琥珀色标签，low 使用灰蓝色标签。
- 表格行高保持紧凑，文本过长时省略并提供 tooltip。

## 文案规范

按钮：

- 生成正式报价
- 运行真实验收
- 打开输出目录
- 选择文件
- 选择目录

状态：

- 空闲
- 正在导入 CAD
- 正在计算算量
- 正在生成报价
- 正在生成复核报告
- 正在校验正式报价包
- 完成
- 失败

错误：

- `验收失败：CAD 导入失败，请检查 DXF 文件和单位设置。`
- `正式报价包校验失败：单价表未套用，请先运行 check-prices。`
- `输入无效：请选择存在的模板 Excel。`

## 实现前设计检查

在写 PySide6 代码前，建议先产出：

- 主窗口线框图。
- 运行页组件清单。
- 结果页组件清单。
- QSS 色彩和间距 token。

实现后建议做一次设计评审，重点看：

- 文字是否溢出。
- 表格是否能扫读。
- 错误状态是否醒目但不过度刺眼。
- 主流程是否不需要说明文字也能操作。
